# AI Workflow

This project is designed to show practical, responsible AI-assisted frontend development.

## Intended workflow

1. Human defines the scope and quality bar.
2. AI scaffolds the component architecture and first pass.
3. Human reviews browser behavior, interaction quality, and product fit.
4. AI assists with tests, refactors, and accessibility review.
5. Human performs final review before submission.

## Guardrails

- AI should not invent production Trek assets, hidden APIs, or proprietary implementation details.
- All product data in this repo is mock data.
- Final submissions should be reviewed manually.
- Accessibility claims should be validated, not assumed.
- Generated code should be kept simple enough to explain in an interview.

## Example Codex prompts

Review this branch as a frontend engineering take-home. Focus on Vue + TypeScript correctness, accessibility gaps, responsive layout risks, test coverage, and whether the repo demonstrates design-system thinking.
