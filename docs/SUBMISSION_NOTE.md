Vue + TypeScript Trek Frontend Demo

## Short version

Hi Trek hiring team,

I built a focused Vue 3 + TypeScript proof-of-ability demo using Trek's public Electric Bikes product listing page as a design reference. The goal was to demonstrate the work described in the Frontend Engineer role: translating a UX/UI reference into responsive, accessible Vue components; organizing a product-listing interface into reusable pieces; using TypeScript; validating with tests/builds; documenting tradeoffs; and showing a responsible AI-assisted development workflow.

Links:

- Live demo: https://trek-proof-of-ability.vercel.app/
- GitHub repo: https://github.com/adamfsalsa/trek-proof-of-ability
- Implementation PR: https://github.com/adamfsalsa/trek-proof-of-ability/pull/3

A few areas I focused on were Vue 3 component architecture, responsive desktop/mobile layout, WCAG 2.1 AA-minded keyboard and screen-reader support, design-token CSS, product comparison state, Vitest/Testing Library smoke tests, GitHub Actions validation, and documentation for accessibility, performance, image policy, and AI-assisted workflow.

I did not have access to Trek's internal Figma files, CMS data, or production assets, so I used public-page references, screenshots, DOM observations, mock data, and stock/placeholder imagery. In a production Trek environment, I would replace those with approved Figma specs, design-system tokens, CMS-backed product data, and official product imagery.

Thank you for taking a look.
